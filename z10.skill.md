# z10 — AI Agent Design Tool

z10 is a design tool where AI agents edit designs using standard DOM APIs.
You write JavaScript; z10 executes it atomically against the design document.

## Setup

```bash
# Install the CLI
npm install -g z10

# Authenticate
z10 login --token <your-api-token>

# Load a project
z10 project load <project-id>

# (Optional) Load a specific page
z10 page load <page-id>
```

## Commands

| Command | Description |
|---------|-------------|
| `z10 login --token <token>` | Authenticate with z10 server |
| `z10 project load <id>` | Set current project (syncs DOM locally) |
| `z10 page load <id>` | Set current page |
| `z10 dom` | Show compact DOM tree of current page |
| `z10 dom --full` | Show full HTML of current page |
| `z10 exec` | Execute JavaScript from stdin |
| `z10 components` | List registered Web Components |
| `z10 tokens` | List design tokens |

## Executing Code

Pipe JavaScript via stdin to `z10 exec`. The entire code block runs atomically
as a single transaction against a sandboxed copy of the document:

```bash
z10 exec <<'EOF'
const nav = document.getElementById('left-nav');
nav.style.padding = '16px';
const item = document.createElement('div');
item.textContent = 'New Item';
nav.appendChild(item);
EOF
```

**On success**:
```
✓ Executed (txId: 5)
```

**On conflict** (someone edited the same elements concurrently):
```
✗ Execution rejected
  Conflict: {"type":"style-property","nid":"left-nav","property":"padding","manifestTs":3,"liveTs":4}
  Fresh HTML available — subtree was modified concurrently.
```

**On error** (code threw an exception):
```
✗ Execution rejected
  Error: Cannot read properties of null (reading 'style')
```

### How It Works

1. The CLI reads your entire code block from stdin
2. It gets a snapshot of the relevant subtree (with a read ticket for conflict detection)
3. Your code runs in a **sandboxed `document`** — a scoped copy, not the live document
4. If your changes don't conflict with concurrent edits, they commit atomically
5. If there's a conflict, the transaction is rejected and you get fresh HTML to retry

### Sandbox Rules

Your code runs in a restricted environment. The `document` you see is a scoped proxy with:
- `querySelector`, `querySelectorAll`, `getElementById` — query within the subtree
- `createElement`, `createTextNode`, `createDocumentFragment` — create new elements
- Standard DOM mutation methods on queried elements

**Not available**: `window`, `fetch`, `setTimeout`, `require`, `import`, network access.

**Do not modify** these attributes — the transaction will be rejected:
- `data-z10-id` — managed by z10 for node identity
- `data-z10-ts-*` — managed by z10 for version tracking

## DOM API Reference

Use standard Web APIs — z10 provides a full DOM environment:

### Query
```js
document.getElementById('left-nav')
document.querySelector('[data-z10-component="Button"]')
document.querySelectorAll('.nav-item')
element.closest('.container')
element.children
element.parentElement
```

### Create & Mutate
```js
document.createElement('div')
parent.appendChild(child)
parent.insertBefore(newNode, refNode)
parent.removeChild(child)
element.remove()
element.cloneNode(true)
```

### Content
```js
element.textContent = 'Hello'
element.innerHTML = '<span>rich content</span>'
```

### Style
```js
element.style.padding = '8px'
element.style.display = 'flex'
element.style.gap = '12px'
element.style.setProperty('--custom-var', 'value')
```

### Attributes
```js
element.setAttribute('data-z10-intent', 'layout')
element.classList.add('active')
element.id = 'sidebar'
```

## z10 Data Attributes

| Attribute | Purpose | Agent access |
|-----------|---------|--------------|
| `data-z10-id` | Stable node identifier — use to address elements | Read only |
| `data-z10-ts-*` | Version tracking timestamps | Do not touch |
| `data-z10-component` | Component type name | Read/write |
| `data-z10-intent` | Layout intent: `layout`, `decoration`, `content`, `interaction` | Read/write |
| `data-z10-editor` | Editor metadata (internal) | Do not touch |
| `data-z10-agent-editable` | Governance: which nodes the agent may edit | Read only |

**Important**: Address elements by `data-z10-id` (e.g., `document.querySelector('[data-z10-id="nav"]')`).
New elements you create will be assigned `data-z10-id` automatically on commit.

## Design Tokens

Set design tokens via the `z10` global:

```js
z10.setTokens('semantic', {
  '--color-primary': '#3b82f6',
  '--color-surface': '#f8fafc',
  '--spacing-md': '16px',
});

z10.setTokens('primitives', {
  '--blue-500': '#3b82f6',
  '--gray-50': '#f8fafc',
});
```

Reference tokens in styles:
```js
element.style.color = 'var(--color-primary)';
element.style.padding = 'var(--spacing-md)';
```

## Web Components

Define components using standard Web Components:

```js
class ChatBubble extends HTMLElement {
  static observedAttributes = ['variant', 'content'];

  // z10 extension: prop schema for design tool property panel
  static z10Props = {
    variant: { type: 'enum', options: ['sent', 'received'], default: 'received' },
    content: { type: 'string', required: true }
  };

  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() { this.render(); }
  attributeChangedCallback() { this.render(); }

  render() {
    const variant = this.getAttribute('variant') || 'received';
    const content = this.getAttribute('content') || '';
    this.shadowRoot.innerHTML = `
      <style>
        .bubble { padding: 12px; border-radius: 16px; }
        .sent { background: var(--color-primary); color: white; }
        .received { background: var(--color-surface); }
      </style>
      <div class="bubble ${variant}">${content}</div>
    `;
  }
}
customElements.define('chat-bubble', ChatBubble);
```

Instantiate components:
```js
const bubble = document.createElement('chat-bubble');
bubble.setAttribute('variant', 'sent');
bubble.setAttribute('content', 'Hello!');
document.getElementById('chat-container').appendChild(bubble);
```

### z10Props Schema

`static z10Props` is optional. When provided, the design tool shows a property panel for the component:

| Type | Schema |
|------|--------|
| `string` | `{ type: 'string', required?: boolean }` |
| `number` | `{ type: 'number', min?: number, max?: number }` |
| `boolean` | `{ type: 'boolean', default?: boolean }` |
| `enum` | `{ type: 'enum', options: string[], default?: string }` |
| `color` | `{ type: 'color', default?: string }` |

## Common Patterns

### Bulk modify
```js
for (const btn of document.querySelectorAll('[data-z10-component="Button"]')) {
  btn.style.opacity = '0.5';
  btn.style.cursor = 'not-allowed';
}
```

### Insert at position
```js
const nav = document.getElementById('left-nav');
const newItem = document.createElement('div');
newItem.className = 'nav-item';
newItem.textContent = 'New Order';
newItem.style.padding = '8px 12px';
nav.insertBefore(newItem, nav.children[2]);
```

### Build a layout
```js
const container = document.createElement('div');
container.style.display = 'flex';
container.style.gap = '16px';
container.style.padding = '24px';

const sidebar = document.createElement('aside');
sidebar.style.width = '240px';
sidebar.setAttribute('data-z10-intent', 'layout');

const main = document.createElement('main');
main.style.flex = '1';

container.appendChild(sidebar);
container.appendChild(main);
document.body.appendChild(container);
```

### Conditional updates
```js
const cards = document.querySelectorAll('.card');
for (const card of cards) {
  const price = parseFloat(card.querySelector('.price')?.textContent || '0');
  if (price > 100) {
    card.style.border = '2px solid var(--color-primary)';
    card.querySelector('.badge')?.remove();
    const badge = document.createElement('span');
    badge.textContent = 'Premium';
    badge.style.background = 'var(--color-primary)';
    badge.style.color = 'white';
    badge.style.padding = '2px 8px';
    badge.style.borderRadius = '4px';
    card.prepend(badge);
  }
}
```

## Error Recovery

### Conflict (concurrent edit)
```
✗ Execution rejected
  Conflict: {"type":"children","nid":"left-nav","manifestTs":3,"liveTs":5}
  Fresh HTML available — subtree was modified concurrently.
```

**Fix**: Another user or agent edited the same elements. Refresh and retry:
```bash
z10 dom          # get fresh DOM state
# re-read the DOM, adjust your code if needed, retry
z10 exec <<'EOF'
// updated code based on fresh DOM
EOF
```

The CLI handles retry with backoff automatically for transient conflicts.

### Element not found
```
✗ Execution rejected
  Error: Cannot read properties of null (reading 'style')
```

**Fix**: Check the DOM for correct IDs:
```bash
z10 dom          # see the tree
```

### Illegal attribute modification
```
✗ Execution rejected
  Error: Illegal modification of data-z10-id or data-z10-ts-* attribute
```

**Fix**: Don't modify `data-z10-id` or `data-z10-ts-*` attributes. These are managed by z10.

### Syntax error
```
✗ Execution rejected
  Error: Unexpected token (line 3, col 12)
```

**Fix**: Check your JavaScript syntax.

## Governance

z10 enforces edit permissions via `data-z10-agent-editable`:
- **full-edit** (default): Agent can edit everything
- **scoped-edit**: Agent can only edit nodes with `data-z10-agent-editable="true"` or descendants
- **propose-approve**: Agent writes to staging branch, human reviews

If governance blocks an edit, you'll see:
```
✗ Execution rejected
  Error: GOVERNANCE_DENIED — node 'header' is not agent-editable
```
